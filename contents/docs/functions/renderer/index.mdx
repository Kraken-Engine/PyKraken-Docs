## Present
<div className="api-card">
<ApiSig name="present" returns="None" />

Present the rendered content to the screen.
This finalizes the current frame and displays it.
Should be called after all drawing operations for the frame are complete.

</div>

## Set Virtual Resolution
<div className="api-card">
<ApiSig name="set_virtual_resolution" params={[{ name: "width", type: "int" }, { name: "height", type: "int" }]} returns="None" />

Set a virtual resolution for rendering. This creates an internal render target of the specified size,
and all rendering will be done to that target, which is then scaled up to the actual screen resolution when presented.

_**Args**_

- `width` : The width of the render target in pixels.
- `height` : The height of the render target in pixels.

_**Raises**_

- `ValueError` : If width or height are not positive integers.

</div>

## Unset Virtual Resolution
<div className="api-card">
<ApiSig name="unset_virtual_resolution" returns="None" />

Unset any previously configured virtual resolution and restore rendering directly to the output.

</div>

## Draw
<div className="api-card">
<ApiSig name="draw" multiline params={[{ name: "texture", type: "Texture" }, { name: "transform", type: "Transform", default: "..." }, { name: "anchor", type: "Vec2", default: "..." }, { name: "pivot", type: "Vec2", default: "..." }]} returns="None" />

Render a texture.

_**Args**_

- `texture` : The texture to render.
- `transform` : The transform (position, rotation, scale).
- `anchor` : The anchor point (0.0-1.0). Defaults to top left (0, 0).
- `pivot` : The rotation pivot (0.0-1.0). Defaults to center (0.5, 0.5).

---

<ApiSig name="draw" multiline params={[{ name: "texture", type: "Texture" }, { name: "dst", type: "Rect" }]} returns="None" />

Render a texture stretched into a destination rectangle.

This is a simpler alternative to the transform-based draw when you only
need to place a texture at a specific screen rectangle without rotation.
The source region is determined by the texture's clip area.

_**Args**_

- `texture` : The texture to render.
- `dst` : Destination rectangle on screen.

</div>

## Draw 9Slice
<div className="api-card">
<ApiSig name="draw_9slice" multiline params={[{ name: "texture", type: "Texture" }, { name: "dst", type: "Rect" }, { name: "slice", type: "Rect" }, { name: "anchor", type: "Vec2", default: "..." }, { name: "pivot", type: "Vec2", default: "..." }]} returns="None" />

Render a texture using 9-slice scaling (9-grid).

This divides the texture into 9 regions: 4 corners (unscaled), 4 edges (scaled in one axis),
and 1 center (scaled in both axes).

_**Args**_

- `texture` : The source texture.
- `dst` : The destination rectangle on screen.
- `slice` : A rectangle defining the slice widths/heights (left_width, top_height, right_width, bottom_height).
- `anchor` : The anchor point. Defaults to top left.
- `pivot` : The rotation pivot. Defaults to center.

</div>

## Draw Batch
<div className="api-card">
<ApiSig name="draw_batch" multiline params={[{ name: "texture", type: "Texture" }, { name: "transforms", type: "Sequence[Transform]" }, { name: "anchor", type: "Vec2", default: "..." }, { name: "pivot", type: "Vec2", default: "..." }, { name: "clip_rects", type: "Sequence[Rect] | None", default: "None" }]} returns="None" />

Render a texture multiple times with different transforms in a single batch call.

This is much faster than calling draw() in a loop because it avoids
per-call Python/C++ dispatch overhead.

_**Args**_

- `texture` : The texture to render.
- `transforms` : A list of transforms (position, rotation, scale).
- `clip_rects` : Per-instance clip rectangles. If provided, these override
        the texture's clip area for each instance. If None, all instances use the texture's clip area.
- `anchor` : The anchor point (0.0-1.0). Defaults to top left (0, 0).
- `pivot` : The rotation pivot (0.0-1.0). Defaults to center (0.5, 0.5).

---

<ApiSig name="draw_batch" multiline params={[{ name: "texture", type: "Texture" }, { name: "transforms", type: "NDArray[numpy.float64]" }, { name: "anchor", type: "Vec2", default: "..." }, { name: "pivot", type: "Vec2", default: "..." }, { name: "batcher", type: "Batcher | None", default: "None" }]} returns="None" />

Render a texture multiple times using a NumPy array for maximum throughput. This is *the* fastest way to render large batches of sprites,
being significantly faster than the list-based draw_batch() due to no-copy viewing of contiguous array data.

Each row of the array describes one instance. The number of columns determines
the layout:

- **2 columns** ``[x, y]`` — position only (angle=0, scale=1).
- **3 columns** ``[x, y, angle]`` — position + rotation (scale=1).
- **4 columns** ``[x, y, angle, scale]`` — position + rotation + uniform scale.
- **5 columns** ``[x, y, angle, scale_x, scale_y]`` — full transform.
- **9 columns** ``[x, y, angle, scale_x, scale_y, clip_left, clip_top, clip_width, clip_height]`` — full transform + per-instance clip rect.

_**Args**_

- `texture` : The texture to render.
- `transforms` : float64 array with shape ``(N, 2|3|4|5|9)``.
- `anchor` : The anchor point (0.0-1.0). Defaults to top left (0, 0).
- `pivot` : The rotation pivot (0.0-1.0). Defaults to center (0.5, 0.5).
- `batcher` : Pre-allocated rendering buffer for higher performance.

_**Raises**_

- `ValueError` : If the array does not have 2, 3, 4, 5, or 9 columns.

</div>

## Read Pixels

<ApiSig name="read_pixels" params={[{ name: "src", type: "Rect", default: "..." }]} returns="PixelArray" />
